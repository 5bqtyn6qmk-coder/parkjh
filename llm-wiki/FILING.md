# 대화·인사이트를 위키에 남기기 (Filing)

훅·자동 스크립트 없이, **채팅 한 번**으로 위키를 고치는 최소 절차다. 형식·역할의 상세는 항상 [`WIKI_SCHEMA.md`](./WIKI_SCHEMA.md)를 따른다.

---

## 사용자 — 이렇게만 요청한다

1. Cursor에서 이 저장소(또는 `llm-wiki/`)가 열린 상태로 채팅·에이전트를 연다.
2. 컨텍스트로 **`@llm-wiki`** 또는 **`WIKI_SCHEMA.md`** 를 붙인다 (규칙·스키마가 잡히게).
3. 아래 같은 말로 **범위**를 적는다.

**트리거 예시 (한국어·영문 아무거나)**

- `위키에 남겨`
- `filing`
- `위키 filing`
- `방금 대화 요약해서 위키에 반영해`

**함께 적으면 좋은 것**

- 넣을 **주제 폴더** (예: `pages/ai/`) 또는 새 주제 이름
- 남길 **내용의 종류** (새 노트 / 기존 `[[슬러그]]` 페이지 수정 / 소스 파일 추가 여부)

예:

> `@llm-wiki` 위키에 남겨. 방금 우리가 말한 깨달음만 bullet로 `pages/general/`에 새 노트 만들고, ingest 로그에 기록해.

---

## 에이전트 — 트리거가 오면 할 일

사용자가 위 트리거로 **위키에 반영**하라고 하면:

1. **`WIKI_SCHEMA.md`** 를 준수한다 (`sources/` 불변, `pages/`만 본문, 파일명 전역 유일 등).
2. 반영 내용을 **`pages/<주제>/`** 에 새 파일 또는 기존 페이지 수정으로 정리한다. Frontmatter·`[[슬러그]]` 규칙 유지.
3. 원문 대화 전체를 `sources/`에 넣지는 **않는다** — 사용자가 “이 텍스트를 소스로 추가해”라고 명시한 경우에만 `sources/`에 새 파일 추가 가능.
4. **`_meta/ingest-log.md`** 에 시간순으로 블록을 append한다. 한 줄 설명에 `filing` 이라고 적어 구분해도 된다.
5. 새 주제·허브가 생기면 **`_index.md`** 에 링크를 추가한다.

민감한 개인정보·비밀키는 위키에 쓰지 않는다. 애매하면 사용자에게 한 줄 확인한다.
