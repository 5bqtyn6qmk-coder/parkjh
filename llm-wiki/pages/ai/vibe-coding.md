# 바이브 코딩

**Karpathy가 널리 인용되는 영문 표현:** *“fully give in to the vibes, embrace exponentials, and forget that the code even exists.”* — 2차 정리·출처는 [Wikipedia — Vibe coding](https://en.wikipedia.org/wiki/Vibe_coding), 원 트윗은 직접 확인 권장.

## 요지

- 자연어로 의도를 주고, 모델·도구가 만든 코드를 **빠르게 돌려 보며** 맞춘다.
- 에러 메시지·로그를 **그대로** 넘겨 수정 루프를 돌리는 패턴이 반복적으로 언급된다.
- **주말·버리기 쉬운 프로토타입**에는 속도·재미가 크고, **장기 유지·안전**이 중요하면 같은 방식만으로는 부족하다는 뉘앙스가 소스에 정리되어 있다.

## 핵심 (활용)

- 프로토타입 vs 프로덕션을 **구분**한다.
- 코드베이스가 커지면 **읽기·검증** 비중을 다시 늘린다.
